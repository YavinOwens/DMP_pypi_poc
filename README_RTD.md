# Read the Docs Setup

This package is configured for automatic documentation hosting on Read the Docs.

## Data Solution

The package functionality focuses on the today's enterprise landscape,
organisations face increasingly complex data management challenges that require sophisticated technologysolutions. The augmented benchmarking tool covers 15 distinct propritery tests revealing the current state of AI model performance, where: These fundamentals are automatically included in Word document reports generated by the LLM when a database session is provided.

**Data Solution Structure:**

The GenAI-Agent solution follows a structured format that includes:

- **Situation**: Describes the current state of data management challenges and context
- **Problem**: Identifies specific problems organizations face
- **Implication**: Explores consequences of not addressing problems
- **Need-payoff**: Presents the value proposition and benefits

The data solution generates products using the best-performing model from the benchmarks and automatically incorporates:

- Data-driven insights from benchmark results
- Cost optimisation recommendations
- Quality improvement metrics
- ROI and business value statements
- Professional consultative language suitable for executives

## Setup Instructions

1. **Sign up for Read the Docs**:

   - Go to https://readthedocs.org/
   - Sign up or log in with your GitHub account
2. **Import the Project**:

   - Click "Import a Project"
   - Select your GitHub repository: `YavinOwens/DMP_pypi_poc`
   - Name: `datamanagement-genai` (or your preferred name)
   - Click "Create"
3. **Configure Build Settings**:

   - The `.readthedocs.yaml` file is already configured
   - Read the Docs will automatically detect it
   - Python version: 3.11
   - Build command: Automatic (uses Sphinx)
4. **Build Documentation**:

   - Click "Build version" to trigger the first build
   - Documentation will be available at: `https://dmp-pypi-poc.readthedocs.io/en/latest/index.html`

## Local Documentation Build

To build documentation locally:

```bash
# Install documentation dependencies
pip install -e ".[docs]"

# Build HTML documentation
cd docs
make html

# View documentation
open _build/html/index.html
```

Or using Sphinx directly:

```bash
pip install -e ".[docs]"
cd docs
sphinx-build -b html . _build/html
```

## Updating Documentation

1. Edit files in the `docs/` directory
2. Commit and push changes
3. Read the Docs will automatically rebuild the documentation

### Key Features

- **Model Benchmarking**: Test and compare Snowflake Cortex AI models
- **Data Quality Management**: LLM-powered data quality rules generation
- **RAG System**: Retrieval-Augmented Generation with multi-backend support
- **Multi-Backend Support**: Snowflake, Qdrant, ChromaDB, or FAISS vector stores
- **Report Generation**: Automated Word document generation with data solution propositions
- **Jupyter Support**: Seamless integration with Jupyter notebooks

### Multi-Backend RAG Support

The package supports multiple vector database backends:

- **Snowflake**: Cloud-native, integrated with Snowflake infrastructure
- **Qdrant**: High-performance vector database
- **ChromaDB**: Easy-to-use, embedded vector database
- **FAISS**: Facebook AI Similarity Search for large-scale similarity search

Each backend can be configured via config files or environment variables. See the main documentation for detailed configuration examples.

```bash
pip install datamanagement-genai[qdrant,chromadb,faiss]

# All features
pip install datamanagement-genai[rag,jupyter,qdrant,chromadb,faiss,local-embeddings,benchmark]
```

## Troubleshooting

If builds fail:

- Check the build logs in Read the Docs dashboard
- Ensure all dependencies are listed in `pyproject.toml` under `[project.optional-dependencies]`
- Verify `.readthedocs.yaml` syntax is correct
- Ensure the package version in `docs/conf.py` matches `pyproject.toml`
